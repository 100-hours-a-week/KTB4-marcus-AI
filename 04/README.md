# 1. 결과를 망가트려보고 싶었다

이번 위클리 프로젝트는 가상 데이터셋을 만들어서  
여러 머신러닝 기법을 적용시켜 결과를 보는 것이였는데,  
여러 예시에서 매번 결과가 어느 정도 잘 나오는 것만 봤지,  
아예 대놓고 망가트리는 것이 가능할까 생각했다.  

이왕이면 아예 위클리 챌린지에서 나온 대부분의 머신러닝 기법들에서 고장나게 해보고 싶었다.  
선형적인 추론을 망가트리고자 한다면, 아마 가장 망가트리기 좋은 방법은 0 하고 1의 무한 반복일 것이다.  
  
```python
# 샘플 데이터 생성
data = {
    'feature1': range(1, 101),  # feature1은 1부터 100까지의 숫자
    'feature2': range(101, 201),  # feature2는 101부터 200까지의 숫자
    'label': [1 if x % 2 == 0 else 0 for x in range(1, 101)]  # label은 짝수이면 1, 홀수이면 0
}
df = pd.DataFrame(data)  # 데이터프레임 생성
```

교재 배포- 인공지능(CNN)의 Dataset Split (데이터셋 분할)에서 나온 예시이다.  
선형적이진 않지만, 그냥 단순 반복이라 특정 추론은 금세 규칙을 발견할 것도 같았다  
그래서 내가 봤던 이상한 함수를 조금 변형하여 0하고 1을 반복하게 하려고 한다.  

바로 칸토어 함수이다.  
수학적으로 정의된 규칙은 있지만, 말로 쉽게 설명하자면  
[0,  1] 중 [1/3,  2/3]를 1/2로,  
나머지 [0,  1/3]과 [2/3, 1] 중 또 중간 1/3 구간을 1/4, 3/4를 주는 것을 반복하는 것이다  

여기서 홀수 번째로 값이 주어지는 구간을 1, 짝수 번째로 값이 주어지는 구간을 0으로 할 것이다.

*이 함수의 특성 상 그냥 함수 그 자체를 정답 데이터로 넣는 것도 좋았겠지만,

숫자가 좀 더 이쁘게 떨어지기 위해 0하고 1 사이를 729 + 1로 `np.linspace` 하려고 했으나,

MLP를 사용할 때 메모리가 버티지 못하여 243 + 1로 분할하였다.

이후 `XI, YI = np.meshgrid(index_x, index_y)`와

`label = XI ^ YI`를 이용하여 칸토어 함수의 값을 만들었다.

## 결과는?

우선 KNN은 항상 정확도가 0.8 이상이 나왔다. 실패했다!

지금 생각해보니 반불규칙적으로 0하고 1을 만든건 좋았는데,

[1/3, 2/3] 구간이 통째로 같은 값이 되었다! **KNN에게 유리할 수 밖에 없었다**

그렇다면 Random Forest는?

그냥 할 때 0.89 정도 나오고, 튜닝된 후에는 0.86 정도가 나온다

아무래도 **범위 별로 값이 나뉘기 때문에 Random Forest에게 유리할 것으로 추정되었다**

다행히도 **SVM**은 버티지 못했다. 정확도 0.5보다도 낮게 나오기도 한다

당연한 일이다. **평면을 구하는데 어떻게 이런 진동하는 0과 1의 데이터셋을 구분할 수 있겠는가**

**Naive Bayes**는 XOR 계산의 특성 상 feature1이랑 feature2를 동시에 고려해야 하기 때문에

**독립을 가정해서는 좋은 결과를 내기 어려워보였다**. 실제로 정확도는 0.5 근처로 머물렀다

활성화 함수가 없는 **퍼셉트론** 역시 평면을 구할 뿐이다.

좋은 결과를 낼 수 없다. 정확도 0.5 미만.

아마 0하고 1을 아예 번갈아서 했으면 정확도가 더 잘 안 나오게 할 수 있었을 것이다.

## 그렇다면 더욱 많은 활성화 함수를 배치한다면?

이제 MLP를 써볼 차례다.

```python
class SimpleNN(nn.Module):
  def __init__(self):
    super(SimpleNN, self).__init__()
    self.fc1 = nn.Linear(2, 128)
    self.fc2 = nn.Linear(128, 64)
    self.fc3 = nn.Linear(64, 32)
    self.fc4 = nn.Linear(32, 1)
    self.sigmoid = nn.Sigmoid()
  def forward(self, x):
    x = torch.relu(self.fc1(x))
    x = torch.relu(self.fc2(x))
    x = torch.relu(self.fc3(x))
    x = self.sigmoid(self.fc4(x))
    return x
```

MLP 테스트 데이터 정확도: 0.6599

정확도가 크지는 않지만 어느 정도 개선된 편이다!

# 4. CNN

이왕이면 CNN도 가우스 함수를 이용하여 해보고 싶다. 어떻게 할까?

필자는 가우스 함수에서 depth를 6까지만 살펴보고,

depth에 3을 나눠서 나머지를 기준으로 색깔을 다르게 배치하기로 하였다.

이번엔 자연수 격자를 만들어야 하니, `feature1`과 `feature2`를 0부터 243까지로 하자

(, 244, 244, 3)의 사진 데이터를 200개 만드는데,

사진의 `사진[:, :, :, 0]`이면

depth를 3으로 나눠서 0인 (x, y) 좌표는 2/3 이상의 랜덤 수

0이 아닌 (x, y) 좌표는 1/3 이하의 랜덤 수를 배치하는 것이다

제미나이의 도움을 좀 받았는데, 우선 2/3 이상과 1/3 이하의 numpy 배열들을 만들고,

label이 0, 1, 2인 것을 boolean으로 만든 numpy 배열을 만들어,

`np.where` 함수를 이용해 조건문처럼 각 `[:, :, :, n]`에 위치하도록 하였다!

그리고 아예 무작위인 데이터 200개와 이런 칸토어 데이터 200개를 만들어

train/test로 분할하여 학습 및 검증을 실행했다.
